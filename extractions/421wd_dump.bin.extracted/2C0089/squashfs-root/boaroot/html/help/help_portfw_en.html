Port Forwarding Feature allows Lan-side hosts to be accessed from Internet (Note: External Port can not be special ports using for some services: tr069, http, https...).

WAN Interface: WAN Interface must be the interface using for Internet service (Note: As soon as you enable NAT feature in this WAN interface, Port Forwarding can work properly)

Port Forwarding Table: Each row is corresponding to each Port Forwarding rule:

  • Mapping name: This field interfies one rule in Port forwarding table.
  • Protocol: This is the protocol use for this rule.
  • Internal IP : This is a IP Address of the device connected in LAN-side network.
  • Internal Port: This is internal Port of this devices in LAN-side network.
  • External Port: This is external port in WAN-side network (Internet) for ONT to have a mapping with internal port. Note: This port is excluded from the list of special port.
  • Status: Enable or disbale the choosen rule.
  • Modify: This feature help us modify currnent parameters.
  • Besides, in the beginning of row, there is a checkbox to choose the rules which need to be erased when we click button "DELETE".

Add form/ Edit form: only display above Port Forwarding when clicking "ADD" button, or below the row which need to be modified when clicking "Modify" Button. In this form, users can choose:

  • Customize: Configures the Port Forwarding parameters (Internal Port, External Port, Internal IP address, Mapping Name, Protocol).
  • Select Application:Allow users to choose some specific service (DNS Server, FPT Server, IPSEC,...). The configuration will be automatically generated in Service Type, Protocol, Mapping Name, Internal port, External Port and users only need to fill IP Address field
